# 12 网关测试 & 文档

## 目标

对飞书和企业微信网关进行完整测试，并编写接入文档，确保网关层稳定可靠。

## 子任务

### 12.1 单元测试

测试消息解析逻辑，无需真实平台连接：

- 飞书私聊消息解析
- 飞书群聊 @机器人 过滤
- 飞书话题群 conversationId 生成
- 企业微信消息解析
- 图片附件提取
- 心跳消息过滤

### 12.2 集成测试

使用 Mock WebSocket 服务器模拟平台推送：

- 消息从网关到 Dispatcher 的完整链路
- 流式响应从 Agent 到网关的完整链路
- 断线重连后消息不丢失
- 并发消息 Lane Queue 串行处理

### 12.3 飞书接入文档

编写 `docs/FEISHU_SETUP.md`，内容包括：

1. 在飞书开放平台创建应用
2. 配置机器人权限（im:message、im:message.group_at_msg）
3. 开启事件订阅（im.message.receive_v1）
4. 获取 appId、appSecret、encryptKey、verificationToken
5. 填写 `~/.friclaw/config.json`
6. 启动服务验证

### 12.4 企业微信接入文档

编写 `docs/WECOM_SETUP.md`，内容包括：

1. 在企业微信管理后台创建智能助手
2. 选择 API 模式 → 长连接方式
3. 获取 botId 和 secret
4. 填写配置并启动

### 12.5 错误处理验证

- 配置缺失时给出明确错误提示
- 网络断开时自动重连，不崩溃
- 平台 API 限流时退避重试
- 消息解析失败时记录日志并跳过

### 12.6 性能测试

- 模拟 10 个并发用户同时发消息
- 验证 Lane Queue 隔离效果
- 测量消息从接收到开始处理的延迟（目标 < 100ms）

## 验收标准

- 单元测试覆盖率 > 80%
- 飞书和企业微信接入文档可独立操作完成
- 断线重连测试通过
- 并发测试无消息乱序
