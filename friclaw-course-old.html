🎯 Dispatcher - 消息路由核心

Dispatcher 是 FriClaw 的中央调度器，负责接收消息并路由到正确的处理器。

核心代码解析

export class Dispatcher { private laneQueue = new LaneQueue() private accepting = true async dispatch( message: Message, reply?: (content: string) => Promise<string>, streamHandler?: StreamHandler ): Promise<void> { // 1. 检查是否接受新消息 if (!this.accepting) throw new Error('Dispatcher is not accepting') // 2. 处理命令消息 (/clear, /new, /status) if (message.type === 'command') { await this.handleCommand(message, reply) return } // 3. 获取或创建会话 const session = this.sessionManager.getOrCreate( message.platform, message.chatId, message.userId, ) // 4. 入队处理 - 保证同一会话串行执行 await this.laneQueue.enqueue(session.id, () => this.agent.handle(session, message, reply, streamHandler) ) } }

🔑 关键特性

特性 说明
Lane Queue 每个会话 ID 对应一个独立队列，确保消息按顺序处理
优雅关闭 停止接受新消息 → 等待队列清空 → 清理资源
命令处理 支持 /clear、/new、/status 等内置命令
流式支持 可选的 streamHandler 实现实时响应

🚦 Lane Queue - 优雅的并发控制

传统的 Mutex 锁需要手动管理，容易出错。FriClaw 使用 Lane Queue 实现更优雅的并发控制。

为什么不用 Mutex？

对比项 Mutex Lane Queue
语义 锁的获取和释放 任务入队和执行
代码复杂度 需要 try-finally 包裹 自动处理异常
执行顺序 不保证 FIFO 严格 FIFO
错误处理 手动释放锁 自动清理

Lane Queue 工作原理

class LaneQueue { private lanes = new Map<string, Promise<void>>() async enqueue<T>(laneId: string, task: () => Promise<T>): Promise<T> { // 获取当前 lane 的最后一个任务 const prev = this.lanes.get(laneId) ?? Promise.resolve() // 创建新任务，等待前一个任务完成 const next = prev.then(task, task) // 更新 lane this.lanes.set(laneId, next.catch(() => {})) return next } }

核心思想：每个 lane (会话 ID) 维护一个 Promise 链，新任务追加到链尾，自动串行执行。

🔥 热重载机制

// 每个会话启动时动态加载 MCP 配置 async startSession(conversationId: string) { // 1. 读取用户自定义 MCP 配置 const userMcp = await loadUserMcpConfig() // 2. 合并内置 MCP 服务 const mergedMcp = { ...builtinMcpServers, ...userMcp } // 3. 写入工作空间 await writeFile(`${workspace}/.mcp.json`, mergedMcp) // 4. 启动 Claude Code 子进程 spawn('claude-code', ['chat', '--workspace', workspace]) }

为什么选择 Bun？

  • ⚡ 极速启动 - 比 Node.js 快 3-4 倍
  • 🔋 内置电池 - SQLite、WebSocket、测试框架开箱即用
  • 📦 兼容 Node.js - 无缝迁移现有代码
  • 🎯 TypeScript 原生支持 - 无需额外配置
# Bun 常用命令 bun run src/index.ts # 运行应用 bun test # 运行测试 bun install # 安装依赖 bun build src/index.ts # 构建应用
