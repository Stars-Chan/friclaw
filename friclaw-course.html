代码透视：调度器的核心方法

左边是 dispatcher.ts 中 dispatch() 方法的真实代码，右边是逐行中文解读。

dispatcher.ts
async dispatch(
  message: Message,
  reply?: (content: string) => Promise<string>,
  streamHandler?: StreamHandler
): Promise<void> {
  if (!this.accepting)
    throw new Error(
      'Dispatcher is not accepting'
    )

  const session =
    this.sessionManager.getOrCreate(
      message.platform,
      message.chatId,
      message.userId
    )

  await this.laneQueue.enqueue(
    session.id,
    () => this.agent.handle(
      session, message,
      reply, streamHandler
    )
  )
}
逐行解读
1-4 定义 dispatch 方法，接收三条信息：消息本体、可选的回复函数、可选的流式处理器。
5-7 检查调度器是否还在工作。如果已经关闭，就直接报错，不再接收新消息。
8-12 通过 会话管理器查找或创建一个会话。用平台 + 聊天ID + 用户ID 来标识唯一会话。
13-18 把任务放入 车道队列。当轮到这个任务时，让 代理 去处理消息。
Aha!

调度器模式是软件工程中最重要的模式之一：把"谁来做"和"做什么"分开。

你只需要把消息交给调度器，不需要关心最终是哪个代理来处理 — 调度器会帮你安排好一切。就像餐厅里的服务员，你点菜后不需要亲自去厨房找厨师。

每个角色都有自己专属的职责，就像一支分工明确的团队。

1
Gateway 网关
接收来自飞书、企微、微信的消息，就像酒店的不同入口。每个入口通向同一个大厅（调度器），但门口的保安（网关）各自负责自己平台的客人。
2

消息路由中心，确保每条消息到达正确的处理者。就像酒店前台，根据客人的需求把信息转给对应的部门。
3
SessionManager
管理对话上下文，每个聊天窗口都是独立的会话。就像酒店给每位客人建立一个独立的档案袋，互不干扰。

5

三层记忆系统：身份、知识、经历，让 AI 越来越懂你。就像一个贴心的管家，记得你的喜好、习惯和历史。
6
CronScheduler
定时任务调度器，在指定时间自动执行任务。就像管家设置的闹钟，到点就提醒你该做什么事。

邮局系统

把 FriClaw 想象成一个高效的邮局系统：

  • 网关是不同颜色的邮箱 — 蓝色收飞书信，绿色收企微信
  • 调度器是分拣室 — 根据地址把信件分发到正确的路线
  • 会话管理器是信格编号 — 确保同一收信人的信件放到同一格

Design

这种把职责分给专门角色的模式叫做"关注点分离"（Separation of Concerns）。每个组件只做一件事，做好一件事。这样当需要修改或扩展时，只需要动一个地方，不会牵一发动全身。

代码透视：车道队列的入队方法

LaneQueue 是 FriClaw 并发控制的核心。理解它的 enqueue 方法，就理解了消息为什么能安全有序地处理。

utils/lane-queue.ts
enqueue<T>(
  laneKey: string,
  task: Task<T>,
  timeoutMs = 300_000
): Promise<T> {
  return new Promise<T>(
    (resolve, reject) => {
      let lane = this.lanes.get(laneKey)
      if (!lane) {
        lane = {
          queue: [],
          running: false
        }
        this.lanes.set(laneKey, lane)
      }
      lane.queue.push({
        task: withTimeout(
          task, timeoutMs
        ),
        resolve, reject
      })
      if (!lane.running)
        this.drain(laneKey)
    }
  )
}
逐行解读
1-3 方法定义：laneKey 是车道标识（即会话 ID）；task 是要执行的任务；timeoutMs 是超时时间，默认 5 分钟。
4-6 返回一个 Promise，这样调用者可以等待任务完成后拿到结果。
7-8 查找是否已有这条车道。如果没有，就创建一条新车道。
9-11 初始化新车道：空队列，不在运行中。
12-14 把任务放入车道队列。用 withTimeout 包裹任务，确保超时后自动取消。
15-16 如果车道没有在运行，立即开始排空队列。
重点 同一车道的任务严格按入队顺序执行（FIFO），不同车道之间可以并发执行。
🚗
高速公路收费系统

把 LaneQueue 想象成高速公路收费站：

  • 每条车道（session）一次只处理一辆车（message）
  • 同一条车道上的车必须排队等候，先到先过
  • 不同车道可以同时通过，互不影响
  • 每辆车有过站时间限制（超时机制），不能永远停在收费站
Tip

超时机制是防御性编程的重要手段。默认 5 分钟超时意味着如果一个 AI 回复卡住了（比如网络问题），最多等 5 分钟就会自动取消，不会永远阻塞后面的消息。这就像收费站给每辆车设了一个最长过站时间。

MemoryManager 初始化过程

Initialize the memory system: create knowledge and episodes directories, open the SQLite database, and instantiate the three memory modules.

async init(): Promise<void> {
  const { dir } = this.config
  // 创建 knowledge 和 episodes 子目录
  mkdirSync(join(dir, 'knowledge'), { recursive: true })
  mkdirSync(join(dir, 'episodes'), { recursive: true })

  // 初始化 SQLite 数据库（含 FTS5 全文搜索表）
  this.db = initDatabase(join(dir, 'index.sqlite'))

  this.identity = new IdentityMemory(this.db, dir)
  this.knowledge = new KnowledgeMemory(this.db, dir)
  this.episode = new EpisodeMemory(this.db, dir)

  log.info({ dir }, 'Memory system initialized')
}

全文搜索 — 用 FTS5 搜索记忆

搜索是记忆系统的核心能力。SQLite 的 FTS5 模块提供了高性能的全文搜索，支持按分类过滤和按相关度排序。

提示：全文搜索(FTS5)是搜索引擎的核心技术，让你像用百度一样搜索本地数据。它通过建立索引来加速搜索。

Search the memory index. If a category filter is provided, restrict results to that category. Results are ranked by relevance. Returns an empty array on any error.

export function search(
  db: Database, query: string, category?: string, limit = 10
): SearchResult[] {
  try {
    if (category) {
      return db.prepare(
        `SELECT id, category, title, content, tags
         FROM memory_fts
         WHERE memory_fts MATCH ? AND category = ?
         ORDER BY rank LIMIT ?`
      ).all(query, category, limit) as SearchResult[]
    }
    return db.prepare(
      `SELECT id, category, title, content, tags
       FROM memory_fts
       WHERE memory_fts MATCH ?
       ORDER BY rank LIMIT ?`
    ).all(query, limit) as SearchResult[]
  } catch { return [] }
}

术语表

全文搜索 (FTS5)：SQLite 内置的全文搜索引擎，通过建立倒排索引，实现对大量文本的快速关键词搜索。
索引：一种数据结构，用于加速数据库查询操作，类似于书籍的目录。
数据库 (SQLite)：轻量级文件型关系数据库，所有数据存储在单个文件中，无需独立服务器。
Markdown (.md)：轻量级标记语言，用纯文本格式编写文档，支持标题、列表、链接等格式。
前缀：在搜索中，前缀指词语的开头部分。FTS5 支持前缀搜索，输入词的开头即可匹配相关结果。

模块五：AI 大脑 — Claude Code 是如何工作的

ClaudeCodeAgent 是 FriClaw 的"大脑"。它通过启动 Claude Code CLI 子进程来处理用户的每一条消息，并将 AI 的回复流式传回。

ClaudeCodeAgent 处理流程

从收到用户消息到返回 AI 回复，一共经历以下 7 个步骤：

1

收到消息，构造 JSON 请求体

2

检查是否有可复用的子进程（进程池）

3

如果没有空闲子进程，启动新的 Claude Code CLI 子进程

4

通过 stdin 发送 JSON 格式的消息

5

从 stdout 逐行读取流式 JSON 响应

6

解析事件类型：thinking_delta、text_delta、tool_use、done

7

将响应回传给调用者（网关或 Dashboard）

stream() 方法核心代码

这是 ClaudeCodeAgent 最核心的方法。它使用 AsyncGenerator 实现流式输出，让调用者可以实时接收 AI 的思考过程和回复。

Stream events from Claude Code: get or create a reusable subprocess, send the user message as JSON via stdin, then read JSON events line-by-line from stdout, yielding thinking, text, and tool-use events until the final result.

async *stream(request: RunRequest): AsyncGenerator<AgentStreamEvent> {
  // 获取或创建可复用的子进程
  const processState = await this.getOrCreateProcess(request)
  const payload = JSON.stringify({
    type: 'user',
    message: { role: 'user', content: buildContent(request) },
  }) + '\n'

  // 通过 stdin 发送消息
  const stdin = processState.proc.stdin
  stdin.write(payload)

  // 逐行读取 stdout 的流式 JSON 响应
  for await (const line of readLines(proc.stdout)) {
    const event = JSON.parse(line)
    if (event.type === 'assistant') {
      // 逐步产出：思考过程、文本回复、工具调用
      for (const block of content) {
        if (block.type === 'thinking') yield { type: 'thinking_delta', text }
        if (block.type === 'text') yield { type: 'text_delta', text }
        if (block.type === 'tool_use') yield { type: 'tool_use', name, input }
      }
    }
    if (event.type === 'result') {
      yield { type: 'done', response: { text, model, costCny, ... } }
    }
  }
}

MCP 协议 — 工具扩展

MCP（Model Context Protocol）是 Anthropic 推出的标准化工具调用协议。它让 Claude 能够调用自定义工具，如读写记忆、管理定时任务等。

每次启动新的 Claude Code 子进程时，FriClaw 会将 MCP 配置写入工作空间的 .mcp.json 文件中，子进程自动加载这些工具。

Write the merged MCP server configuration into the workspace directory so the Claude Code subprocess can discover and use the available tools.

writeFileSync(
  join(workspaceDir, '.mcp.json'),
  JSON.stringify({ mcpServers }, null, 2),
  'utf-8'
)
比喻：打电话 — 你拨号（spawn 创建子进程），通话（JSON 流式传输），挂断（dispose 释放进程）。
性能优化：子进程复用 + 空闲超时是性能优化的经典模式 — 不用每次都重新启动。空闲超过 5 分钟的子进程会自动销毁。

术语表

子进程 (Subprocess)：由父进程创建的独立运行程序。FriClaw 主进程通过 Bun.spawn 创建 Claude Code CLI 子进程。
标准输入 (stdin)：进程接收数据的通道。主进程通过子进程的 stdin 写入 JSON 数据。
标准输出 (stdout)：进程发送数据的通道。Claude Code 通过 stdout 逐行输出 JSON 事件流。
流式传输 (Streaming)：数据像水流一样持续不断地小块传输。AI 回复可以边生成边展示。
进程池 (Process Pool)：复用已完成任务的子进程，避免重复启动开销。
协议 (Protocol)：计算机之间通信的规则和格式约定。MCP 是 Anthropic 定义的工具调用协议。

CronScheduler 调度器核心代码

schedule() 方法是调度器的核心。它支持两种模式：标准 cron 表达式（循环任务）和 ISO 时间字符串（一次性任务）。

Schedule a cron job. If the expression is an ISO datetime, treat it as a one-shot timer. Otherwise, normalize the 5-field cron expression to 6-field format (adding a seconds prefix) and register it with the Cron library using the specified timezone.

private schedule(job: CronJob): void {
  // 检查是否为 ISO 时间字符串（一次性任务）
  if (this.isISODateTime(job.cronExpression)) {
    this.scheduleOneShot(job); return
  }

  // 标准化 cron 表达式：5 字段转 6 字段
  let expression = job.cronExpression
  const fields = expression.trim().split(/\s+/)
  if (fields.length === 5) {
    expression = `0 ${expression}`
  }

  // 使用 Cron 库注册定时任务
  const cron = new Cron(expression, { timezone }, () => {
    this.fire(job.id)
  })
  this.schedulers.set(job.id, cron)
}

startDaemon() — 系统启动主流程

这是 FriClaw 的入口函数。它按顺序初始化所有组件，并将它们连接在一起。

比喻：智能家居系统 — 所有设备（组件）通过中枢（调度器）连接，统一控制。

Start the FriClaw daemon: load configuration, initialize the memory system, create the session manager and AI agent, wire up the dispatcher, start the cron scheduler, launch all enabled gateways, and register graceful shutdown handlers.

async function startDaemon(): Promise<void> {
  const config = await loadConfig()

  // 初始化三层记忆系统
  const memory = new MemoryManager(config.memory)
  await memory.init()

  // 创建会话管理器
  const sessionManager = new SessionManager({
    workspacesDir, timeoutMs
  })

  // 创建 AI 代理（注入身份记忆作为系统提示）
  const agent = new ClaudeCodeAgent({
    model, allowedTools, soulContent: memory.identity.read()
  })

  // 创建调度器（连接会话管理器和 AI 代理）
  const dispatcher = new Dispatcher(sessionManager, agent, async () => {
    await agent.dispose()
    await memory.shutdown()
  })

  // 启动定时任务调度器
  const cronScheduler = new CronScheduler(dbPath)
  await cronScheduler.start()

  // 启动所有网关（飞书、企微、微信等）
  const gateways = [
    new FeishuGateway(cfg),
    new WecomGateway(cfg)
  ]
  await Promise.all(gateways.map(g => g.start(dispatcher)))

  // 注册优雅关闭处理器
  registerShutdownHandlers(dispatcher)
}
关键设计：优雅关闭是生产级应用的必备特性 — 先停止接收新消息，再处理完队列中的消息，最后清理资源。

术语表

守护进程 (Daemon)：在后台持续运行的程序。FriClaw 的 startDaemon() 函数启动守护进程模式。
优雅关闭 (Graceful Shutdown)：先停止接收新请求，处理完进行中的任务，最后释放资源。通过监听 SIGTERM/SIGINT 信号触发。
配置文件：存储程序运行参数的文件。FriClaw 使用 config.json 存储各组件的配置。
验证 (Validation)：检查数据是否符合预期格式。FriClaw 使用 Zod 库进行配置验证。
环境变量：操作系统提供的键值对存储，用于向程序传递配置信息，如 API 密钥。
